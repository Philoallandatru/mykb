# kv-cache-quantization

## 定义

KV Cache Quantization 是将 KV cache 从高精度（FP16/BF16）压缩到低精度（INT8/INT4）的技术，以减少内存占用和带宽需求。

## 背景：KV Cache 内存问题

### 内存规模
对于批次 512、上下文 2048 的场景：
- **FP16**: KV cache ~3TB
- **相当于**: 约 3 倍模型参数大小
- **瓶颈**: 限制批次大小和上下文长度

### 内存带宽
- KV cache 需频繁从 HBM 加载
- 带宽限制成为推理瓶颈
- 特别是在长上下文场景

## 量化技术

### 1. SmoothQuant (W8A8)

**核心思想**: 将激活中的异常值迁移到权重

**方法**:
1. 分析激活的通道级别统计量
2. 计算平滑因子 s
3. 应用数学等价变换: `Y = (X · diag(s)^-1) · (diag(s) · W)`
4. 激活和权重都量化到 INT8

**优势**:
- 激活和权重同时量化
- 保持数学等价性
- 性能损失小

**应用**:
- LLaMA、OPT、BLOOM 等模型
- 训练后量化，无需重新训练

### 2. GPTQ (3-4 bit 权重)

**核心思想**: 基于二阶信息的逐层量化

**方法**:
1. 使用少量校准数据
2. 逐层应用最优脑量化（OBQ）
3. 量化到 3-4 bits per parameter
4. 补偿量化误差

**性能**:
- **OPT-175B**: 压缩到 3-4 bits
- **性能损失**: 可忽略不计
- **内存节省**: 4-5 倍

**限制**:
- 主要针对权重，不是 KV cache
- 但可结合使用

### 3. KV Cache 专用量化

**Per-Token Quantization**:
- 每个 token 的 KV 独立量化
- 记录 scale 和 zero-point
- 动态量化，适应不同输入

**Per-Channel Quantization**:
- 按 channel 维度量化
- 减少量化参数开销
- 平衡精度和效率

**混合精度**:
- 重要 token (如 attention 峰值) 保持高精度
- 其他 token 使用低精度
- 自适应策略

## 量化格式

### INT8 (8-bit)
- **压缩比**: 2× (相比 FP16)
- **精度损失**: 很小 (< 1%)
- **硬件支持**: 广泛（Tensor Core INT8）

### INT4 (4-bit)
- **压缩比**: 4× (相比 FP16)
- **精度损失**: 轻微 (1-3%)
- **硬件支持**: 部分 GPU

### FP8 (8-bit floating point)
- **压缩比**: 2× (相比 FP16)
- **精度**: 优于 INT8
- **硬件**: H100+ 原生支持

## 实现挑战

### 1. 动态范围
- KV cache 的值域随输入变化
- 需要动态调整量化参数
- 增加运行时开销

### 2. 精度损失累积
- 长上下文场景下误差累积
- 影响注意力计算准确性
- 需要校准和补偿策略

### 3. 硬件支持
- 低精度矩阵乘法支持
- 量化/反量化开销
- 内存访问模式变化

## 性能提升

### 内存节省
- **INT8**: 2× 内存节省
- **INT4**: 4× 内存节省
- **批次大小**: 可加倍或更多

### 吞吐量提升
- 减少内存带宽需求
- 更多并发请求
- 10-30% 吞吐量提升（取决于瓶颈）

### 延迟影响
- 量化/反量化开销: < 5%
- 内存带宽节省抵消开销
- 总体延迟可能降低

## 工程实践

### vLLM 集成
- 支持 KV cache INT8 量化
- 自动校准和量化
- 配置选项控制

### 最佳实践
1. **评估瓶颈**: 确认内存是瓶颈
2. **选择精度**: INT8 通常是最佳起点
3. **校准数据**: 使用代表性数据
4. **监控质量**: 跟踪输出质量指标
5. **A/B 测试**: 对比量化前后效果

## 与其他技术结合

- **Multi-Query Attention**: 先减少 KV cache 量，再量化
- **PagedAttention**: 量化后的块管理
- **Prefix Caching**: 共享量化的前缀 KV
- **Offloading**: 量化后再 offload 到 SSD

## 研究方向

1. **更低精度**: INT2、混合精度
2. **学习量化**: 训练感知量化
3. **自适应**: 根据重要性动态调整精度
4. **硬件协同**: 利用新硬件特性
